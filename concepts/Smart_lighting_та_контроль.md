# Smart lighting та централізоване керування

> Технологічний шар, що дозволяє містам реалізовувати динамічні режими освітлення відповідно до часу, потреби, погоди і інших факторів.

## Архітектура smart-системи

```
┌──────────────────────────────────────────────────────┐
│                    CMS                               │
│      Central Management System (cloud/local)        │
│                                                      │
│   - GIS-інтеграція                                   │
│   - Розклади / графіки                               │
│   - Алармування / faulting                          │
│   - Енергомоніторинг                                 │
└─────────────────┬────────────────────────────────────┘
                  │
                  │   (LoRaWAN / NB-IoT / 4G/5G)
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │ Lamp 1 │ │ Lamp 2 │ │ ...    │
   │ NEMA   │ │ Zhaga  │ │ Custom │
   │ socket │ │ socket │ │        │
   └────────┘ └────────┘ └────────┘
```

## Ключові компоненти

### Світильник з сокетом
- **NEMA socket** — стандарт США/міжнародний для кріплення controllers
- **Zhaga Book 18** — європейський стандарт для модульних D4i drivers
- **D4i** — інтерфейс driver↔controller з поверненим зв'язком (енергія, температура, помилки)

### Controller (модуль на лампі)
- Радіо-зв'язок з gateway
- Локальний таймер
- Сенсори (необов'язково: рух, освітленість, температура)

### Gateway
- Збирає дані від множини controllers
- Передає в CMS через інтернет

### CMS (Central Management System)
- Дашборд з картою світлоточок
- Розклади (на окремий світильник або групу)
- Аларми про несправності
- Аналітика енергоспоживання

## Сценарії використання

### Bi-level (basic)
Два режими: повний (25 lx) увечері та зменшений (12 lx) після опівночі. **Економія: 30-40%** енергії.

### Adaptive (advanced)
- **Time-based**: різні графіки за порами року
- **Detection-based**: PIR/радар детектори підвищують яскравість при русі
- **Weather-based**: збільшення освітленості в дощ/сніг
- **Event-based**: підвищення для свят, парадів

**Економія: 50-70%** при збереженні безпеки.

### Per-fixture personalization (Plan Lumière)
Конкретні світлоточки реалізують спеціальні художні сценарії:
- ілюмінація фасадів за програмою
- відключення опівночі
- сезонні зміни кольору

## Реалізація у досліджуваних містах

### Мадрид
**Найбільший в світі smart-проєкт**: 225 000 точок з керуванням. Philips Interact City. Адаптація яскравості за середовищем і часом.

### Копенгаген
Citelum-керована система, інтегрована з smart-city стратегією.

### Відень
Schreder/інші постачальники, інтеграція з MA 33 операційною системою.

### Інші
Поступове впровадження, часто гібридні системи (старі+нові кварт.)

## Ризики і виклики

### Кібербезпека
Світильники — IoT-пристрої, що можуть стати вектором атак. Важливо:
- Encrypted communication
- Firmware update mechanism
- Network segmentation

### Vendor lock-in
Якщо CMS і controllers одного постачальника — нелегко перейти. **Стандарт D4i + Zhaga знижує цей ризик**.

### Обслуговування
Smart-системи вимагають IT-компетенцій у муніципальних службах. Рішення:
- Аутсорсинг (ESCO)
- Внутрішні навчальні програми
- API для партнерських служб

### Дані і приватність
PIR-сенсори можуть фіксувати рух. Питання збереження і використання цих даних — окрема дискусія.
